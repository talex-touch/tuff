# 智能提示词库

Tuff 中各种 AI 能力的精选高质量提示词集合。

## 快速开始

**导入提示词**

1. 从本页面下载提示词文件
2. 打开 Tuff → 设置 → 智能 → 提示词
3. 点击"导入"并选择下载的 JSON 文件
4. 提示词将在您的提示词库中可用

**使用提示词**

提示词可以通过两种方式使用：

1. **直接使用**：在提示词管理器中选择提示词并直接使用
2. **能力绑定**：在设置 → 智能 → 能力中将提示词绑定到特定能力

**类别**：文本改写  
**描述**：将随意消息转换为专业邮件

## 创建自定义提示词

**提示词结构**

```json
{
  "id": "唯一标识符",
  "name": "显示名称",
  "category": "text|code|analysis|vision|translation",
  "description": "提示词功能的简要描述",
  "content": "带有清晰指令的实际提示词内容",
  "variables": ["var1", "var2"],
  "builtin": false,
  "createdAt": 1703001234567,
  "updatedAt": 1703001234567
}
```

**最佳实践**

1. **清晰的指令**：明确说明您希望 AI 做什么
2. **结构化**：使用标题和格式组织复杂的提示词
3. **示例**：在有帮助时包含示例
4. **上下文**：提供必要的背景信息
5. **约束**：指定任何限制或要求
6. **输出格式**：定义预期的输出结构

**变量**

使用变量使提示词可重用：

```
分析以下 {{language}} 代码：
{{code}}
```

使用提示词时变量会自动替换。

## 提示词工程技巧

**1. 具体明确**

❌ "总结这个"  
✅ "创建一个 3 段摘要，重点关注关键决策和行动项"

**2. 提供上下文**

❌ "审查这段代码"  
✅ "审查这个 TypeScript React 组件的安全问题和性能优化机会"

**3. 定义输出格式**

❌ "分析数据"  
✅ "分析数据并以结构化格式呈现发现：1) 关键洞察，2) 趋势，3) 建议"

**4. 使用示例**

包含期望输出的示例以指导 AI：

```
示例输出：
## 摘要
[简要概述]

## 关键点
- 要点 1
- 要点 2
```

**5. 迭代和优化**

测试您的提示词并根据结果进行优化。小的调整可以显著提高输出质量。

## 故障排除

**提示词未按预期工作**

1. **检查变量**：确保提供所有必需的变量
2. **审查指令**：使指令更具体
3. **使用不同模型测试**：某些提示词在特定模型上效果更好
4. **调整温度**：降低温度以获得更一致的结果

**结果不一致**

1. **添加约束**：更严格地指定输出格式
2. **提供示例**：包含示例输出
3. **使用系统提示词**：设置清晰的角色和期望
4. **多次测试**：确保跨运行的一致性
