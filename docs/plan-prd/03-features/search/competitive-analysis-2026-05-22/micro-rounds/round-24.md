# 微审计 24/70

- 审计主题：Raycast Translate / AI Commands / OCR 链路，是否能映射到 Tuff 当前 `touch-translation`、`touch-intelligence`、Assistant 的真实闭环，而不是只停留在一个“翻译插件”壳。
- 读取/核对的文档或源码锚点：
  - `docs/plan-prd/03-features/search/competitive-analysis-2026-05-22/07-translation-ocr-ai-commands.md`
    - 第 1 节把翻译、OCR、AI 命令收为同一条“输入来源 -> 参数化命令 -> provider 执行 -> 可见结果 -> 可恢复失败”的链路。
    - 第 3.1 / 3.2 / 3.3 节把 `quick-translate-v1`、`clipboard-image-ocr-translate-v1`、`ai-command-template-v1` 切成最小实施切片。
  - `plugins/touch-translation/manifest.json`
    - `touch-translate` 声明 `acceptedInputTypes: ["text"]`，`screenshot-translate` 声明 `acceptedInputTypes: ["text", "image"]`。
    - 权限含 `clipboard.read`、`network.internet`、`intelligence.basic`、`window.create`。
  - `plugins/touch-translation/index/main.ts`
    - `resolveTextToTranslate()` 在 `screenshot-translate` 场景下会从 `inputs` 取 `image` 数据并调用 `vision.ocr`。
    - `translateImageFromClipboardInput()` 走 `image.translate.e2e`，并把 `provider` / `model` / `traceId` 写进结果 payload。
    - `buildWidgetItem()` / `buildImageTranslationItem()` 说明当前已有可见 widget / DivisionBox 结果，而不是纯后台黑盒。
  - `apps/core-app/src/main/modules/assistant/module.ts`
    - `handleScreenshotTranslate()` 通过 `translateClipboardImage()` 打通剪贴板图片翻译，并把 `SCENE_UNAVAILABLE` / `IMAGE_UNAVAILABLE` 归一为可恢复失败。
  - `apps/core-app/src/main/modules/box-tool/core-box/image-translate.ts`
    - `translateClipboardImage()`、`translateCoreBoxImageItem()`、`translateImageBase64()` 形成了从 clipboard / item / path 到 Nexus scene 的统一入口。
  - `apps/core-app/src/main/modules/ai/intelligence-module.ts`
    - 注册了 `text.translate`、`vision.ocr`、`image.translate.e2e` 三个关键 capability。
  - `apps/core-app/src/shared/events/omni-panel.ts`
    - `acceptedInputTypes` 与 `OmniPanelContextPayload` 已经能承接上下文输入，但还不是 Raycast 级用户可编排模板。
- 结论：
  - 这条链路不是空壳。Tuff 已经同时具备：插件侧翻译入口、截图 / 剪贴板图片 OCR、`image.translate.e2e`、CoreBox / Assistant 的统一转发、以及 AI capability 注册。
  - 主文档对“当前有真实散点，但还缺统一合同与 evidence”的判断是准确的。现在能证明的是能力链已通，不足的是统一的 `AiCommandTemplate` / `ProviderEvidence` / `PrivacyRedactionPolicy`，而不是再补一个独立翻译插件。
  - 因此本轮不需要修正主文档；更合理的后续方向仍是把翻译、OCR、AI 命令收进同一个参数与证据模型，而不是扩成完整 Raycast 式 command editor。
