# 微审计 19/70

- 审计主题：Browser Data 作为 Quicklinks / Web Bookmarks / uTools 网页快开数据源时，Tuff 当前 `touch-browser-data` 是否只是“源码里有个插件”，还是已经具备只读书签扫描、来源诊断和隐私边界；以及主文档把它标成“部分落地，需要 evidence”是否准确。
- 读取/核对的文档或源码锚点：
  - `docs/plan-prd/03-features/search/competitive-analysis-2026-05-22/01-basic-capability-alignment.md`
    - 第 3 节把 Browser Data 标为“部分落地，需要 evidence”，锚点是 `plugins/touch-browser-data/index.js` 只读扫描 Chrome / Edge / Brave / Arc Bookmarks JSON，并输出 source diagnostics。
    - 第 4 节把 Quicklinks / Web Search 的当前状态归为分散能力，要求后续 `QuicklinkSource` 复用 `touch-browser-open` 执行与 Browser Data source diagnostics。
    - 第 5 节 P0 要求 Browser Data evidence 覆盖 Chrome / Edge / Brave / Arc Bookmarks JSON、`read-failed` / `not-found` / `unsupported` diagnostics、打开/复制 URL action。
  - `docs/plan-prd/03-features/search/competitive-analysis-2026-05-22/09-cross-platform-local-data.md`
    - 第 3 节确认 Browser Data 首版只读落地，需 `fs.read`，输出 `supported` / `unsupported` / `not-found` / `read-failed` diagnostics。
    - 第 5 节明确浏览器 History、Cookies、Sessions、Login Data、Safari 历史/书签数据库不应默认扫描或同步。
    - 第 6 节把 Browser Data bookmarks evidence 收敛为 `fs.read` denied、available、not-found、read-failed、unsupported、open URL、copy URL。
  - `docs/plan-prd/03-features/search/competitive-analysis-2026-05-22/10-execution-roadmap-synthesis.md`
    - `browser-data-evidence-v1` 明确验收 Chrome / Edge / Brave / Arc Bookmarks JSON 样本、失败态和打开/复制 URL action；同时禁止读 History SQLite、写回浏览器或持久化完整书签 JSON dump。
  - `docs/plan-prd/03-features/search/competitive-analysis-2026-05-22/11-100-round-cross-review-ledger.md`
    - 第 67 条已做一次总账判断：当前只读 Chrome / Edge / Brave / Arc Bookmarks JSON，输出 available / not-found / read-failed / unsupported；文档没有宣称 History / Safari 已落地。
  - `plugins/touch-browser-data/manifest.json`
    - `sdkapi: 260428`、`category: "utilities"`；required permission 只有 `fs.read`，optional permission 是 `clipboard.write`。
    - feature `browser-data` 只接受 text，关键词覆盖 `browser-data` / `browser` / `chrome` / `edge` / `brave` / `arc` / `bookmarks` / `浏览器` / `书签`，三平台均启用。
  - `plugins/touch-browser-data/index.js`
    - `SUPPORTED_BROWSERS = ['chrome', 'edge', 'brave', 'arc']`；macOS / Windows / Linux 分别定义 Chromium profile root，其中 Linux 不包含 Arc，后续 diagnostics 会把 Arc 标成 unsupported。
    - `discoverBookmarkFiles()` 只查 `Bookmarks` 文件与标准 profile 目录，限制每个浏览器最多 8 个 profile；没有读 History、Cookies、Sessions、Login Data。
    - `normalizeUrl()` 只允许 `http:` / `https:` URL，测试里 `file:///tmp/demo.txt` 会被过滤。
    - `scanBrowserBookmarks()` 返回 `{ items, diagnostics }`，diagnostics 会区分 `available`、`not-found`、`read-failed`、`unsupported`，并记录 `profileCount`、`reason`、`lastError`、`failedProfile`。
    - `onFeatureTriggered()` 执行前请求 `fs.read`，拒权时推送“缺少文件读取权限” item，不把拒权伪装成空结果。
    - `buildBookmarkItem()` 的默认动作是 `open-url`，并添加 `copy-url` action；item meta 带 `pluginName`、`featureId`、`actionId` 和 URL payload。
  - `plugins/touch-browser-data/index.test.cjs`
    - 覆盖 query prefix / browser filter、Chromium folder flatten、非 http URL 过滤、profile discovery、read-failed reason、Linux Arc unsupported、ranking、open item + copy URL action、empty state source availability。
- 结论：
  - 主文档判断成立。`touch-browser-data` 不是空壳插件：它已经有 manifest、权限理由、CoreBox feature、跨平台 Chromium Bookmarks 路径、只读扫描、结果构造、打开 URL、复制 URL action 和 focused tests。
  - Browser Data 目前适合被定义为“只读浏览器书签数据源”，而不是完整浏览器数据平台。源码只读取 Chromium `Bookmarks` JSON，并且过滤到 http / https URL；这与主文档“不读 History、不写回浏览器、不持久化完整 JSON dump”的边界一致。
  - 当前实现的 source diagnostics 是后续 Quicklinks 合同的重要资产。它能告诉用户 Chrome / Edge / Brave / Arc 某个来源是 available、not-found、read-failed 还是 unsupported，避免把“没权限、没装浏览器、读坏文件、平台不支持”混成普通无结果。
  - 仍不能把它宣传成 Raycast / Alfred 级完整 Web Bookmarks 或 Browser History 体验。缺口仍包括真机/packaged evidence、Safari 调研、History SQLite 只读副本策略、source disable / clear / rebuild UI、持久索引策略、以及和 QuicklinkSource 的统一管理入口。
  - KISS/YAGNI 角度看，下一步应先补 `browser-data-evidence-v1`，不应直接扩大到浏览器历史或 Safari。理由是当前书签链路已经小而完整，最缺的是证据包；历史和 Safari 会显著提高隐私、权限和数据库兼容复杂度。
- 是否发现需修正的主文档问题：否。现有 `01`、`09`、`10`、`11` 对 Browser Data 的状态描述与源码一致：书签首版已落地但需要 evidence；History / Safari / 持久索引 / 清理 UI 都被正确放在后续，不存在把源码存在夸大成完整产品闭环的问题。
